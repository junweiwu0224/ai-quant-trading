# OpenClaw Integration Plan

## Product Direction

The system will integrate a full OpenClaw service, not a reduced version. OpenClaw acts as the assistant engine and orchestration layer, while this quant platform remains the source of truth for market data, watchlists, paper trading, reports, permissions, and audit records.

Each user gets an independent logical workspace with their own OpenClaw workspace, conversations, skills, memories, paper trading context, reports, and permissions.

## Invite Registration

- Registration requires a 6-character invite code.
- Invite codes are generated by administrators.
- Codes should use uppercase letters and digits, preferably excluding confusing characters such as `O`, `0`, `I`, and `1`.
- Each invite code can define expiration, max uses, default role, default permission template, and default workspace template.
- Store invite code hashes rather than plaintext codes.
- Record every successful and failed invite-code use.

## User And Workspace Model

Default model:

```text
User
  -> Workspace
      -> OpenClaw Workspace
      -> Watchlist
      -> Paper Trading Account
      -> Skills
      -> Memories
      -> Reports
      -> Audit Logs
```

The first version should use logical isolation on top of a shared full OpenClaw service. Dedicated OpenClaw instances can be added later for stronger isolation if needed.

## OpenClaw Service Integration

- Run the full OpenClaw Gateway as a backend sidecar service.
- The project carries `openclaw-2026.5.20.tgz` and the dashboard managed runner can install it into `node_modules` on first use, so local development does not depend on a globally installed `openclaw` binary.
- Managed startup writes a minimal `openclaw.json` under the configured OpenClaw work directory, enables `/healthz`, `/readyz`, `/v1/models`, `/v1/chat/completions`, `/v1/responses`, and starts `openclaw gateway run`.
- Use this quant platform as the business tool provider.
- Do not directly expose live trading actions.
- Expose only approved tools and APIs to OpenClaw.
- Let OpenClaw chat, install skills, use trusted tools, and call this system's business APIs.
- Put OpenClaw settings under the user avatar menu, scoped to the current user workspace.

## Permissions

Suggested permission keys:

```text
chat                General chat
read_market         Market data, stock details, reports, announcements
read_portfolio      Paper trading assets, performance, positions
write_watchlist     Add and remove watchlist items
write_paper_trade   Paper orders, cancellations, closes
manage_skills       Install, enable, and disable skills
manage_workspace    Workspace-level settings
admin               System administration
```

Execution policy:

- Read-only actions can execute directly.
- Watchlist updates can execute directly or with lightweight confirmation.
- Paper trading orders require explicit confirmation.
- Skill installation requires confirmation or a trusted source.
- Live trading stays disabled.

## Audit Logs

Record important actions with:

- User ID
- Workspace ID
- OpenClaw session ID
- Tool name
- Input parameters
- Result status
- Confirmation status
- Source: page, OpenClaw, or automation
- Timestamp

## Daily Paper Trading Report

Daily reports are a core product feature.

Supported modes:

- User manually asks OpenClaw to generate a report.
- System automatically generates a post-market report.
- Historical daily reports can be viewed later.
- Weekly and monthly reports can be added later.

Report contents:

- Total equity
- Daily PnL
- Cumulative return
- Max drawdown
- Today's trades
- Current positions
- Risk events
- Best and worst symbols
- Tomorrow's watchlist
- OpenClaw summary and suggestions

## Personal Research Memory

Research memory should be domain-specific, not just chat history.

Store:

- Why a stock was watched.
- Why a stock was added to watchlist.
- Why a simulated order was placed.
- Loss review notes.
- User strategy preferences.
- User industry preferences.
- User risk style.
- Strategy tuning history.

Example user question:

```text
我之前为什么关注这只股票？
```

OpenClaw should answer from recorded research memory rather than generic market commentary.

## Current Acceptance

OpenClaw is implemented as a full external service with a managed local wrapper. Each user gets one platform workspace, one OpenClaw workspace ID, workspace-scoped tools, and workspace-scoped audit, memory, skill, and daily report records.

The platform exposes only approved quant tools to OpenClaw, and live trading remains disabled. Simulated paper-trading actions require confirmation according to the current workspace settings.
