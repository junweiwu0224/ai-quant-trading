# 0002 Market breadth uses the latest high-coverage date

- 状态：Accepted
- 日期：2026-06-09
- 相关链接：`data/storage/storage.py`, `/api/market/breadth`, `dashboard/static/intelligence-market.js`, `dashboard/static/intelligence-signals.js`

## 背景

市场情绪和市场信号依赖 `stock_daily` 的全市场涨跌广度。实际同步中，最新日可能只
写入少量股票，例如盘后或补采任务先写入 15 只样本。如果直接使用 `MAX(date)`，页面
会把零散样本显示成全市场方向，导致上涨/下跌数量和广度分失真。

用户目标是全量覆盖和可解释可信的市场雷达，而不是追逐最新但覆盖不足的局部样本。

## 决策

我们决定：

- `/api/market/breadth` 的存储层选择最近一个覆盖率达到阈值的交易日，当前阈值为
  `stock_info` 全量股票数的 80%。
- 如果最新日期低于阈值，则忽略该零散最新日，回退到最近高覆盖交易日。
- API 返回 `ignored_latest_date`、`ignored_latest_date_covered`、
  `ignored_latest_date_coverage_pct` 和 `min_selected_date_coverage_pct`，供前端解释。
- 情报页市场情绪和市场信号必须展示使用的交易日、有效/全量样本，以及被忽略的零散最新日。

## 主要取舍

选择高覆盖日期的原因：

- 广度指标是横截面指标，样本覆盖比日期新鲜度更重要。
- 本地数据同步可能分批完成，最大日期不等于可用于全市场判断的日期。
- 前端解释被忽略的零散日期，可以同时保留透明度和可信度。

接受的代价：

- 市场广度在非交易时段或分批同步期间可能显示 T-1 高覆盖日，而不是数据库中的最大日期。
- 80% 阈值是运行时经验阈值，未来可根据同步稳定性或交易所覆盖范围调整。

## 影响

- 正面影响：市场情绪、市场信号、情报页首屏不再被最新少量样本污染。
- 负面影响 / 风险：如果本地库长期没有高覆盖日，接口仍会回退到最大日期并通过覆盖字段暴露风险。
- 需要同步修改的地方：任何新增全市场广度、板块广度或雷达口径，都应优先检查覆盖率再使用日期。

## 后续

- 如果数据同步稳定后需要更严格口径，可把阈值提升到 90% 或使用最近完整交易日状态文件。
- 可以把高覆盖日期选择逻辑抽成共享 helper，供热力图、雷达和其它横截面指标复用。
