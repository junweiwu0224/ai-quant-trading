# Glossary

本文件记录 AI Quant Trading 中容易混淆的业务术语、技术术语和命名约定。

## 业务术语

- A 股：中国大陆股票市场标的。
- 自选股：用户关注的股票列表，常作为数据同步、行情订阅和机会池默认范围。
- 因子：用于预测、筛选或解释股票表现的特征，例如技术、基本面、资金流、情绪、宏观因子。
- Alpha：与市场基准无关的超额收益信号或研究模块。
- 回测：用历史数据验证策略表现。
- 模拟盘：不连接真实券商账户的纸面交易环境。
- 实盘：真实券商账户和真实下单链路，高风险。
- 条件单：条件触发后创建的模拟盘订单意图。
- 机会池：按信号、评分或规则筛出的候选股票列表。
- 风控：仓位、止损、集中度、回撤和权限等风险约束。

## 技术术语

- Dashboard：FastAPI + Jinja2 + Vanilla JS 的 Web 面板。
- Qlib：历史兼容命名；当前部分运行时能力是轻量本地信号，不等同完整 pyqlib 训练栈。
- Signal Engine：运行时 AI 信号边界，详见 `docs/decisions/0001-signal-engine-v2.md`。
- OpenClaw：外部/本地 agent 服务集成，用于工作区、聊天、工具桥接和审计。
- Agentic：Agent 驱动的投研、策略实验、信号市场和模拟盘桥接领域层。
- DataHub：数据健康、覆盖率、决策矩阵和数据源状态相关页面/API。
- Watchlist：自选股。
- E2E：Playwright 浏览器端到端测试。

## 命名约定

- `qlib_*`：legacy 兼容字段，新增代码应确认是否应该使用 Signal Engine 语义。
- `signal_*`：新信号语义优先命名。
- `paper_*`：模拟盘相关。
- `openclaw_*`：OpenClaw 服务、路由、工具或前端模块。
